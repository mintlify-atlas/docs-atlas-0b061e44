# Cardio

Cardio is a lightweight Kotlin library designed to simplify interaction with PostgreSQL databases using Vert.x PG Client reactively and with coroutines.

Built on top of Vert.x PG Client, Cardio provides a clean, idiomatic Kotlin API that embraces reactive programming patterns while maintaining the simplicity and readability that Kotlin developers expect.

## Why Cardio?

Cardio provides a lightweight alternative to traditional blocking JDBC drivers and more complex reactive solutions. It sits on top of Vert.x PG Client, which is a reactive PostgreSQL client based on the PostgreSQL wire protocol, providing:

- **Better Performance**: Non-blocking I/O means your application can handle more concurrent connections with fewer threads
- **Kotlin-First Design**: Built specifically for Kotlin, with coroutines as a first-class citizen
- **Simple API**: Clean, minimal surface area that's easy to learn and use
- **Production Ready**: Built on battle-tested Vert.x technology used in production by thousands of companies

### Comparison to Alternatives

While similar to R2DBC in philosophy (reactive database access), Cardio is specifically designed for PostgreSQL and built on Vert.x rather than R2DBC's SPI. This focused approach allows for:

- Tighter integration with the PostgreSQL protocol
- Simpler API surface without the abstraction overhead of multi-database support
- Direct access to PostgreSQL-specific features when needed
- Better performance through Vert.x's proven event loop architecture

# Quick Start

This guide will walk you through the basics of using Cardio to interact with your PostgreSQL database.

## Setup

<Steps>
  <Step title="Create a Cardio instance">
    First, create an instance of `Cardio` by configuring the PostgreSQL connection and pool options:

    ```kotlin
    import io.github.blad3mak3r.cardio.postgres.Cardio
    import io.vertx.sqlclient.PoolOptions

    val cardio = Cardio.create<Cardio> {
        url("postgres://user:password@localhost:5432/my_database?sslmode=disable")
        poolOptions = PoolOptions().apply {
            maxSize = 10
        }
    }
    ```

    You can also use `PgConnectOptions` for more detailed configuration:

    ```kotlin
    import io.vertx.pgclient.PgConnectOptions

    val cardio = Cardio.create<Cardio> {
        connectOptions = PgConnectOptions().apply {
            host = "localhost"
            port = 5432
            database = "my_database"
            user = "user"
            password = "password"
        }
        poolOptions = PoolOptions().apply {
            maxSize = 10
        }
    }
    ```

    <Note>
      The `create` method uses reflection to instantiate your Cardio instance and automatically verifies the database connection by querying the PostgreSQL version.
    </Note>
  </Step>

  <Step title="Execute queries within transactions">
    Use the `inTransaction` block to safely execute queries. Transactions automatically commit on success or rollback on error:

    ```kotlin
    data class User(val id: Int, val name: String)

    cardio.inTransaction { tx ->
        // Execute a query and map results
        val users = tx.query("SELECT * FROM users WHERE active = $1", listOf(true)) { row ->
            User(
                id = row.getAs<Int>("id"),
                name = row.getAs<String>("name")
            )
        }

        // Execute an update
        tx.execute("UPDATE users SET last_login = NOW() WHERE active = $1", listOf(true))
    }
    ```

    <Tip>
      The `getAs<T>()` extension method provides type-safe column access. Use `getAsNullable<T>()` for nullable columns.
    </Tip>
  </Step>

  <Step title="Create repositories for data access">
    For better organization, extend `CardioRepository` to encapsulate your data access logic:

    ```kotlin
    import io.github.blad3mak3r.cardio.postgres.CardioRepository

    class UserRepository(db: Cardio) : CardioRepository<Cardio>(db) {
        suspend fun findById(id: Int): User? {
            return query("SELECT * FROM users WHERE id = $1", listOf(id)) { row ->
                User(
                    id = row.getAs<Int>("id"),
                    name = row.getAs<String>("name")
                )
            }.firstOrNull()
        }

        suspend fun create(name: String): Long {
            return execute("INSERT INTO users (name) VALUES ($1)", listOf(name))
        }
    }
    ```

    Then use your repository:

    ```kotlin
    val userRepo = UserRepository(cardio)

    // Find a user
    val user = userRepo.findById(1)

    // Create a new user
    val rowsAffected = userRepo.create("John Doe")
    ```

    <Info>
      Repository methods automatically participate in the current transaction context if called within an `inTransaction` block, or create a new connection if called outside.
    </Info>
  </Step>
</Steps>

## Advanced: Custom Cardio Subclasses

The `Cardio` class is `open`, allowing you to extend it for strongly-typed repositories:

```kotlin
// Define your custom database class
class MyDb(pool: Pool) : Cardio(pool)

// Create an instance
val myDb = Cardio.create<MyDb> {
    connectOptions = PgConnectOptions().apply {
        host = "localhost"
        port = 5432
        database = "my_database"
        user = "user"
        password = "password"
    }
    poolOptions = PoolOptions().apply {
        maxSize = 10
    }
}

// Define a repository that requires MyDb
class MyRepo(db: MyDb) : CardioRepository<MyDb>(db) {
    suspend fun customOperation() {
        // Your custom operations here
    }
}

// Use the repository with your custom database
val myRepo = MyRepo(myDb)
```
